# 表单数据无法插入数据库 - 问题排查清单

## ✅ 已修复的问题

### 1. **添加事务提交机制** (最重要)
- **文件**: `MessageServlet.java`
- **修改内容**: 
  - 关闭自动提交: `conn.setAutoCommit(false)`
  - 执行SQL后手动提交: `conn.commit()`
  - 异常时回滚: `conn.rollback()`
- **原因**: 即使executeUpdate返回成功，如果没有显式commit，数据可能未真正写入数据库

### 2. **修正表单action路径**
- **文件**: `contact.html`
- **修改前**: `action="addMsg"`
- **修改后**: `action="/new3/addMsg"`
- **原因**: 使用绝对路径避免上下文路径问题

## ❌ 常见错误及解决方案

### 错误1: ClassNotFoundException: com.mysql.cj.jdbc.Driver
**原因**: MySQL驱动未加载
**解决**: 确保mysql-connector-java.jar在WEB-INF/lib目录下

### 错误2: Access denied for user 'root'@'localhost'
**原因**: MySQL密码错误
**解决**: 修改DBUtil.java中的PWD字段

### 错误3: Table 'web_demo.message' doesn't exist
**原因**: 表未创建
**解决**: 执行上面的CREATE TABLE语句

### 错误4: Column 'studentId' cannot be null
**原因**: 表单字段名为studentId，但数据库字段为studentid
**解决**: 已在代码中处理，确保getParameter("studentId")正确

### 错误5: 404 Not Found
**原因**: Servlet路径配置错误
**解决**: 确认@WebServlet("/addMsg")和表单action="/new3/addMsg"匹配
